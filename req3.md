สร้าง website Research Paper Library (ชื่อ "PaperVault") ซึ่งเป็น platform สำหรับให้นักวิจัย upload งานวิจัย/preprint ขึ้นมาเป็นชุด (paper หนึ่งใบ = หนึ่ง folder ที่มีไฟล์ PDF ตัวเปเปอร์ + ไฟล์ประกอบ เช่น dataset, รูป, โค้ด, bibtex) และ user คนอื่นๆมองเห็น เข้าไปอ่าน detail ภายในหรือกดดาวน์โหลดทั้งชุดได้

มี user ทั้งหมด 2 role คือ 1.user (นักวิจัยทั่วไป) 2.admin (บรรณารักษ์ระบบ)

ระบบ lab (กลุ่มวิจัย) ตัว lab จะมีความสัมพันธ์เป็น many to many กับ user โดย default user จะไม่มี lab อะไรเลย และหาก user อยู่ใน lab ไหน user สามารถมองเห็น paper ของ lab นั้นๆได้แม้ paper นั้นจะถูก set เป็น private ก็ตาม แต่ถ้าเป็นคนที่อัพ paper นั้นเองหรือเป็น admin จะเห็น paper นั้นๆได้เลยทุกใบไม่ต้องสนว่า public/private หรืออยู่ lab ไหน

ในหน้า frontend มีหน้าทั้งหมด 5 หน้าคือ

1. register/login โดยใช้เป็น register และ login แบบ local เอาข้อมูลใส่ database และเทียบ database ตรงๆ แต่มีการ encrypt ข้อมูลนิดนึง (อย่างน้อย hash password และ encrypt ฟิลด์อ่อนไหวเช่น email) โดยใช้ database เป็น filebase หรือ serverless นะ ตอนสมัครให้เก็บชื่อนักวิจัย (display name) ไว้ใช้เป็นชื่อเจ้าของ paper ด้วย

2. หน้า library ให้เป็นหน้าที่ลิส paper ทั้งหมดโดยจะเห็นเป็นการ์ด paper พร้อมชื่อเปเปอร์ ชื่อเจ้าของ field (category เช่น CS, Biology, Physics) และ tag ของ paper นั้นๆ และสามารถกดที่ tag หรือ field เพื่อ filter paper ที่ list ออกมาได้ โดยมันจะ list ออกมาแค่ paper ที่ tag ตรงหรือ field ตรงกับที่เราคลิกเท่านั้น (คลิก tag หรือ field ซ้ำเพื่อเลิก filter) และการ์ด paper จะมีกรอบอยู่ 2 สี สีแดงสำหรับ paper ที่อัพแบบ internal และสีดำสำหรับ paper ที่อัพแบบ external และปุ่มที่ให้กดสำหรับ upload paper จะอยู่ในหน้านี้ด้วย โดยมีการอัพทั้งหมด 2 แบบคือ internal และ external โดยที่ internal คือการให้อัพ paper โดยการอัพโหลด folder/ไฟล์โดยตรง (PDF ตัวเปเปอร์ + ไฟล์ประกอบทั้งหมด) แต่ external คือการให้คนกดแนบลิ้ง GitHub repo (หรือลิ้ง arXiv listing) แล้วระบบทำการ traversal ทุก folder ภายในว่ามี paper ไหม (นับว่าเป็น paper เมื่อ folder นั้นมีไฟล์ PDF หรือไฟล์ metadata เช่น paper.json / abstract.md อยู่) โดยถ้าเจอ 3 paper ก็ให้อัพแยกขึ้นเป็น 3 paper และชื่อเจ้าของ paper ให้ใช้เป็นชื่อของผู้อัพ และในหน้า library ไม่ว่าจะเป็น internal หรือ external ข้อมูลที่เราเก็บจะเป็น folder ทั้งหมด และทางเราจะเก็บทั้ง folder เพื่อให้ตอนกดโหลดได้ file จริง และเมื่อกดที่ paper ไหนก็จะสามารถเข้าไปที่หน้า paper ของ paper นั้นๆได้ ส่วน paper ทุกใบจะมีรูปดาวแปะอยู่ขวาบน หากกดจะถือว่าเรา bookmark สิ่งนั้นและเพิ่มน้ำหนักให้การแนะนำ paper ภายหลัง และเราสามารถ filter เพื่อดูแต่ paper ที่เรากดดาวได้ (ดาวกดซ้ำเพื่อเลิกได้เป็นปุ่ม toggle) ลืมบอก paper ที่ระบบแนะนำ กับ paper ที่ลิสทั้งหมดนี่คือแบ่งเป็นคนละโซนนะ ในหน้านี้ให้มันแสดงทั้งคู่แหละ

3. หน้า paper จะเห็น file ทั้งหมดของ folder ของ paper นั้นๆว่ามี file อะไรบ้าง (PDF, รูป, dataset, bibtex ฯลฯ) และสามารถกดเข้าไปดูเนื้อหาของ file นั้นๆได้ (ไฟล์ข้อความเปิดดูได้ตรงๆ ส่วน PDF แสดง preview ได้) และสามารถกด download ลงมาได้ แต่การกด download จะให้ทั้ง folder เลยไม่สามารถเลือกเฉพาะ file ได้ และหากคนที่กดเข้ามาดูเป็นคนที่โพส paper นั้นหรือเป็น admin จะสามารถแก้ไขเนื้อหาของ paper (เช่น abstract / metadata) เพิ่มลด tag หรือ field ได้

4. timeline เป็นหน้าแสดง chart โดยจะมีทั้งหมด 5 chart ดังนี้ 1. paper upload overtime แสดงเป็นกราฟเส้น (กดให้กลายเป็นกราฟแท่งได้) โดยบอกปริมาณ paper ที่ถูกอัพวันนั้น และแบบสะสมตลอด lifetime 2. recent upload แสดง 5 paper ที่พึ่งถูกอัพโหลดตรงๆ 3. paper distribution by field กราฟวงกลมพร้อมบอกอัตราส่วน paper ว่าอยู่ใน field ใดบ้างพร้อมบอกเปอร์เซ็นต์ 4. top tag บอกชื่อ tag ที่มีในระบบมากที่สุดพร้อมจำนวน paper มา 10 tag 5. กราฟวงกลมเหมือน paper distribution by field แต่รอบนี้บอกเป็นอัตราส่วน internal กับ external แทน

5. tag/field management 1. โดยทุกคนสามารถ add tag/field ให้ paper ที่เป็นเจ้าของได้ (ถ้าไม่ใช่เจ้าของจะมองไม่เห็นปุ่มแก้) แต่ถ้าเป็น admin จะทำได้ไม่สนว่าเป็นเจ้าของไหม 2. ถ้าเป็น admin สามารถสร้างหรือลบ tag / field / lab ของระบบได้เลย (user ปกติมองไม่เห็นหน้านี้)

6. permission 1. admin สามารถเพิ่มสมาชิกเข้า lab ต่างๆได้ 2. ทุกคนสามารถปรับ paper เป็น public และ private ได้ ถ้าเป็น private สามารถเพิ่ม lab ที่ให้มองเห็นได้เลย (เฉพาะเจ้าของ paper นั้นๆหรือ admin เท่านั้น)
